# 接口调用流程说明文档

## 示例接口：用户登录 `/api/user/login`

本文档以用户登录接口为例，详细说明从前端调用到数据库查询的完整流程。

---

## 1. 前端调用方式

前端通过 HTTP POST 请求调用该接口：

```javascript
// 前端代码示例（使用 fetch）
const login = async (username, password) => {
  const response = await fetch('http://localhost:9999/api/user/login', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      username: 'zhangsan',
      password: '123456'
    })
  });
  
  const result = await response.json();
  // result 格式：
  // {
  //   "code": 200,
  //   "message": "success",
  //   "data": {
  //     "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  //     "userInfo": { id: 1, username: "zhangsan", ... }
  //   }
  // }
  
  if (result.code === 200) {
    // 保存 token 到本地存储
    localStorage.setItem('token', result.data.token);
    // 跳转到首页
  }
};
```

### 请求信息
- **URL**: `http://localhost:9999/api/user/login`
- **方法**: `POST`
- **Content-Type**: `application/json`
- **请求体**: 
```json
{
  "username": "zhangsan",
  "password": "123456"
}
```

## 2. 程序运行流程

### 步骤 1：请求到达 Controller 层

**文件位置**: `src/main/java/com/health/controller/UserController.java`

```java
@PostMapping("/login")
public Result<Map<String, Object>> login(@Valid @RequestBody LoginRequest request) {
    try {
        Map<String, Object> data = userService.login(request);
        return Result.success(data);
    } catch (RuntimeException e) {
        return Result.error(e.getMessage());
    }
}
```

**说明**：
- `@PostMapping("/login")` 注解匹配 POST 请求到 `/api/user/login` 路径
- `@RequestBody` 将 JSON 请求体自动转换为 `LoginRequest` 对象
- `@Valid` 注解触发参数校验（验证 `@NotBlank` 等注解）
- 调用 `UserService.login()` 方法处理业务逻辑
- 使用 `Result.success()` 统一封装返回结果

### 步骤 2：参数校验（LoginRequest DTO）

**文件位置**: `src/main/java/com/health/dto/LoginRequest.java`

```java
@Data
public class LoginRequest {
    @NotBlank(message = "用户名不能为空")
    private String username;

    @NotBlank(message = "密码不能为空")
    private String password;
}
```

**说明**：
- `@NotBlank` 注解确保用户名和密码不能为空
- 如果校验失败，Spring 会自动返回 400 错误，不会进入 Controller 方法

### 步骤 3：调用 Service 层（业务逻辑）

**文件位置**: `src/main/java/com/health/service/UserService.java`

```java
public Map<String, Object> login(LoginRequest request) {
    // 3.1 查询用户
    User user = userMapper.selectOne(
            new LambdaQueryWrapper<User>().eq(User::getUsername, request.getUsername()));

    // 3.2 验证用户是否存在
    if (user == null) {
        throw new RuntimeException("用户不存在");
    }

    // 3.3 验证密码
    if (!user.getPassword().equals(encryptPassword(request.getPassword()))) {
        throw new RuntimeException("密码错误");
    }

    // 3.4 生成 JWT Token
    String token = jwtUtil.generateToken(user.getId(), user.getUsername());

    // 3.5 清除敏感信息
    user.setPassword(null);

    // 3.6 构建返回结果
    Map<String, Object> result = new HashMap<>();
    result.put("token", token);
    result.put("userInfo", user);
    return result;
}
```

**详细流程**：

1. **查询用户**: 通过 `userMapper.selectOne()` 方法查询数据库
2. **验证用户**: 检查用户是否存在，不存在则抛出异常
3. **验证密码**: 使用 BCrypt 校验前端通过 HTTPS 提交的原始密码与数据库哈希
4. **生成 Token**: 使用 JWT 工具类生成包含用户信息的 token
5. **清除敏感信息**: 将用户对象中的密码字段设为 null，避免返回给前端
6. **返回结果**: 返回包含 token 和用户信息的 Map

### 步骤 4：Mapper 层（数据访问）

**文件位置**: `src/main/java/com/health/mapper/UserMapper.java`

```java
@Mapper
public interface UserMapper extends BaseMapper<User> {
}
```

**说明**：
- `UserMapper` 继承了 `BaseMapper<User>`，自动拥有了基础的 CRUD 方法
- `userMapper.selectOne()` 方法实际执行的 SQL 语句：

```sql
SELECT * FROM user 
WHERE username = 'zhangsan' 
AND deleted = 0 
LIMIT 1
```

**MyBatis-Plus 自动处理**：
- `LambdaQueryWrapper` 构建的查询条件会自动转换为 SQL WHERE 子句
- `@TableLogic` 注解会自动在查询条件中添加 `deleted = 0`（逻辑删除）
- 查询结果自动映射到 `User` 实体对象

### 步骤 5：数据库查询

执行 SQL 查询，从 `user` 表中查询匹配的用户记录。

**数据库表结构**（参考 `schema.sql`）：
```sql
CREATE TABLE `user` (
  `id` BIGINT NOT NULL AUTO_INCREMENT,
  `username` VARCHAR(50) NOT NULL,
  `password` VARCHAR(128) NOT NULL,
  `nickname` VARCHAR(50) DEFAULT '',
  ...
  `deleted` TINYINT DEFAULT 0,
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_username` (`username`)
)
```

### 步骤 6：密码哈希与验证

**文件位置**: `src/main/java/com/health/service/UserService.java`

```java
private final BCryptPasswordEncoder passwordEncoder = new BCryptPasswordEncoder();

private String encryptPassword(String password) {
    return passwordEncoder.encode(password);
}
```

**说明**：
- 前端不执行固定盐 MD5，生产传输必须使用 HTTPS
- 注册时由后端 BCrypt 生成随机盐哈希并存储
- 登录时使用 `passwordEncoder.matches(rawPassword, storedHash)` 校验

### 步骤 7：生成 JWT Token

**文件位置**: `src/main/java/com/health/util/JwtUtil.java`

```java
String token = jwtUtil.generateToken(user.getId(), user.getUsername());
```

**说明**：
- JWT Token 包含用户 ID 和用户名信息
- Token 有效期配置在 `application.yml` 中（默认 7 天）
- Token 用于后续接口的身份认证

### 步骤 8：返回响应

**文件位置**: `src/main/java/com/health/vo/Result.java`

```java
public static <T> Result<T> success(T data) {
    Result<T> result = new Result<>();
    result.setCode(200);
    result.setMessage("success");
    result.setData(data);
    return result;
}
```

## 4. 异常处理

如果登录失败（用户不存在或密码错误），Service 层会抛出 `RuntimeException`：

```java
// UserController.java
catch (RuntimeException e) {
    return Result.error(e.getMessage());
}
```

**返回的错误响应**：
```json
{
  "code": 500,
  "message": "用户不存在"  // 或 "密码错误"
}
```

**前端处理**：
```javascript
const result = await response.json();
if (result.code !== 200) {
    alert(result.message);  // 显示错误信息
    return;
}
```

## 5. 其他接口的认证流程

登录接口不需要认证，但其他需要认证的接口（如 `/api/user/info`）需要携带 Token：

### 5.1 需要认证的接口示例

**文件位置**: `src/main/java/com/health/controller/UserController.java`

```java
@GetMapping("/info")
public Result<User> getUserInfo(HttpServletRequest request) {
    Long userId = (Long) request.getAttribute("userId");
    User user = userService.getUserInfo(userId);
    return Result.success(user);
}
```

### 5.2 前端调用方式

```javascript
// 获取用户信息（需要认证）
const getUserInfo = async () => {
  const token = localStorage.getItem('token');
  
  const response = await fetch('http://localhost:9999/api/user/info', {
    method: 'GET',
    headers: {
      'Authorization': 'Bearer ' + token  // 登录时获取的 token
    }
  });
  
  const result = await response.json();
  return result.data;
};
```

### 5.3 JWT 拦截器验证流程

**文件位置**: `src/main/java/com/health/config/JwtConfig.java`

```java
class JwtInterceptor implements HandlerInterceptor {
    @Override
    public boolean preHandle(HttpServletRequest request, HttpServletResponse response, Object handler) {
        // 1. 从请求头获取 Authorization
        String authHeader = request.getHeader("Authorization");
        
        // 2. 验证格式（必须是 "Bearer token"）
        if (authHeader == null || !authHeader.startsWith("Bearer ")) {
            response.setStatus(401);
            return false;
        }
        
        // 3. 提取 token
        String token = authHeader.substring(7);
        
        // 4. 验证 token 有效性
        if (!jwtUtil.validateToken(token)) {
            response.setStatus(401);
            return false;
        }
        
        // 5. 从 token 中提取 userId，放入 request 属性
        Long userId = jwtUtil.getUserIdFromToken(token);
        request.setAttribute("userId", userId);
        
        return true;  // 放行请求
    }
}
```

**拦截器配置**：
```java
registry.addInterceptor(new JwtInterceptor())
    .addPathPatterns("/api/**")  // 拦截所有 /api/** 路径
    .excludePathPatterns(
        "/api/user/login",      // 排除登录接口
        "/api/user/register",   // 排除注册接口
        "/api/system/**"        // 排除系统接口
    );
```

## 6. 项目架构总结

### 6.1 分层架构

```
┌─────────────────────────────────────┐
│  Controller 层（接口层）             │
│  处理 HTTP 请求，参数校验，返回响应  │
└──────────────┬──────────────────────┘
               │
┌──────────────▼──────────────────────┐
│  Service 层（业务逻辑层）            │
│  处理业务逻辑，调用 Mapper            │
└──────────────┬──────────────────────┘
               │
┌──────────────▼──────────────────────┐
│  Mapper 层（数据访问层）             │
│  定义数据访问接口                    │
└──────────────┬──────────────────────┘
               │
┌──────────────▼──────────────────────┐
│  MyBatis-Plus（ORM 框架）            │
│  自动生成 SQL，执行数据库操作         │
└──────────────┬──────────────────────┘
               │
┌──────────────▼──────────────────────┐
│  MySQL 数据库                        │
│  存储数据                            │
└─────────────────────────────────────┘
```

### 6.2 数据流向

```
HTTP 请求 
  → Controller（接收请求，参数校验）
    → DTO（请求参数对象）
      → Service（业务逻辑处理）
        → Mapper（数据访问接口）
          → MyBatis-Plus（生成 SQL）
            → MySQL（执行查询）
              → Entity（实体对象）
                → Service（业务处理）
                  → VO（响应对象）
                    → Controller（封装 Result）
                      → HTTP 响应（JSON）
```

### 6.3 关键组件说明

| 组件 | 作用 | 位置 |
|------|------|------|
| **Controller** | 接收 HTTP 请求，调用 Service，返回响应 | `controller/` |
| **Service** | 业务逻辑处理，事务管理 | `service/` |
| **Mapper** | 数据访问接口定义 | `mapper/` |
| **Entity** | 数据库表映射实体 | `entity/` |
| **DTO** | 数据传输对象（请求参数） | `dto/` |
| **VO** | 视图对象（响应数据） | `vo/` |
| **Config** | 配置类（CORS、JWT、MyBatis-Plus） | `config/` |
| **Util** | 工具类（JWT 工具） | `util/` |

## 8. 总结

这是一个标准的 Spring Boot + MyBatis-Plus 分层架构项目：

1. **前端**通过 HTTP 请求调用后端接口
2. **Controller** 层接收请求，进行参数校验
3. **Service** 层处理业务逻辑
4. **Mapper** 层定义数据访问接口
5. **MyBatis-Plus** 自动生成 SQL 并执行
6. **数据库**存储和查询数据
7. 结果通过 **Result** 统一封装返回给前端

整个流程清晰、职责分明，符合单一职责原则和分层架构设计。
