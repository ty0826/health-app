# 前端运行与打包说明

## 环境要求

- Node.js 22 或更高版本
- pnpm 11（项目通过 Corepack 固定版本）
- 微信开发者工具：微信小程序预览和发布
- Android Studio + Android SDK：本地 Android 原生运行
- macOS + Xcode：本地 iOS 原生运行和签名
- Expo/EAS 账号：云端 Android/iOS 构建

启用 pnpm：

```bash
corepack enable
pnpm install
```

## API 地址

复制 `.env.example` 为本地 `.env`，按运行平台设置地址：

```text
TARO_APP_API_BASE_URL=http://localhost:9999/api
EXPO_PUBLIC_API_BASE_URL=http://192.168.1.100:9999/api
```

微信开发者工具和 H5 可以使用 `localhost`。Android/iOS 模拟器或真机不能把 `localhost` 当作电脑，需要使用电脑局域网 IP 或可访问的 HTTPS 域名。

登录和注册密码不会在前端执行 MD5 或其他固定哈希。生产 API 必须使用 HTTPS，密码只在后端使用 BCrypt 哈希存储。

## 微信小程序

开发监听：

```bash
pnpm dev:weapp
```

生产构建：

```bash
pnpm build:weapp
```

使用微信开发者工具导入 `frontend/dist`。

## H5

开发监听：

```bash
pnpm dev:h5
```

生产构建：

```bash
pnpm build:h5
```

产物位于 `frontend/dist`，可以部署到任意静态网站服务。

## React Native 公共 Bundle

开发监听：

```bash
pnpm dev:rn
```

生产 Bundle：

```bash
pnpm build:rn
```

该命令使用 Metro 生成 Android/iOS 共用的 `dist/index.bundle`。RN 构建不会使用 Vite。

## Android

首次使用 EAS：

```bash
pnpm exec eas login
pnpm exec eas init
```

开发客户端：

```bash
pnpm dev:android
```

云端构建 APK/AAB：

```bash
pnpm build:android
```

本地原生运行需要提前安装 Android Studio、Android SDK，并配置模拟器或连接真机。

## iOS

开发客户端（需要 macOS）：

```bash
pnpm dev:ios
```

云端构建 IPA：

```bash
pnpm build:ios
```

iOS 发布需要 Apple Developer 账号、Bundle ID 和签名证书。Windows 可以生成并验证 RN Bundle，但不能使用 Xcode 本地生成 IPA。

## 质量检查

```bash
pnpm test
pnpm typecheck
pnpm build:weapp
pnpm build:h5
pnpm build:rn
```

RN 转换目前会提示部分 Web CSS 属性在原生端被忽略，例如 `transition`、伪类和 `backdrop-filter`。这些是兼容性警告，不阻止 Bundle 生成；需要逐页提升原生视觉一致性时，可以继续增加 `.rn.scss` 平台样式。
